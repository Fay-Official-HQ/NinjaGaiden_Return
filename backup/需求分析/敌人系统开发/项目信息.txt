## 三、核心子系统详解

### 3.1 组件化设计（Component-Based）

这是整个项目最核心的架构决策。玩家(`Player`)采用**组件解耦模式**，将不同职责拆分为独立子节点：

| 组件 | 文件 | 职责 |
|------|------|------|
| [InputComponent](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/components/InputComponent.gd) | `InputComponent.gd` | 轮询键盘输入，提供跳跃缓冲(0.12s)，统一暴露 `move_direction` / `jump_pressed` 等接口 |
| [MovementComponent](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/components/MovementComponent.gd) | `MovementComponent.gd` | 纯物理职责：移动、停止、跳跃、重力施加重力 `700`，跳跃力 `260` |
| [AnimationComponent](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/components/AnimationComponent.gd) | `AnimationComponent.gd` | 封装 `AnimatedSprite2D`，带容错检查（动画名不存在时不会崩溃），防重复播放 |
| [SwordComponent](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/components/SwordComponent.gd) | `SwordComponent.gd` | TP 资源系统（最大16），5种剑术的冷却管理（普通5s/必杀10s），输入缓冲(0.3s)支持连招 |
| [NinjutsuComponent](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/components/NinjutsuComponent.gd) | `NinjutsuComponent.gd` | MP 资源系统（最大16），4种忍术工厂方法，切换逻辑 |

**关键设计模式：** 组件通过 `get_parent().owner` 获取 Player 引用，状态机通过 `owner as Player` 回引。

### 3.2 状态机系统（Finite State Machine）

[StateMachine](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/player/states/StateMachine.gd) 实现了经典FSM模式：

- **自动收集子节点**：在 `_ready()` 中遍历所有 `State` 子类作为可用状态
- **企业级兜底**：如果忘记拖拽初始状态，自动回退到 `IdleState`
- **生命周期钩子**：`enter(msg) → update(delta) → physics_update(delta) → exit()`
- **消息传递**：`change_state(new_state, {"key": value})` 支持带参状态切换

**19 个状态完整列表：**

```
基础运动:  Idle → Run → Jump → Fall → Hurt
强化运动:  Crouch → PlatformDrop → Hang → Wall
攻击系统:  GroundAttack → CrouchAttack → AirAttack
忍术系统:  GroundNinjutsu → AirNinjutsu
剑术系统:  SwordReady → SwordDash → SwordUppercut → SwordDownslash → SwordSpin
必杀系统:  DragonFlash
```

### 3.3 战斗伤害系统

采用 **HitBox/HurtBox 双层架构**：

```
┌─────────────┐     area_entered     ┌─────────────┐
│  AttackHitBox │ ──────────────────→ │   HurtBox    │
│  (玩家攻击)   │   take_damage()     │  (敌人受击)   │
│  monitoring=true                    │  emit signal │
└─────────────┘                      └──────┬───────┘
                                            ↓
            ┌──────────┐            ┌──────────────┐
            │ EnemyHitBox│ ────────→│ Player.HurtBox│
            │ (敌人攻击) │          │  (玩家受击)    │
            └──────────┘            └──────────────┘
```

**继承层次：**
- `AttackHitBox`（基类）→ `PlayerHitBox`（普攻，击中回1TP）、`SwordHitBox`（剑术）、`DragonFlashHitBox`（必杀）
- `HurtBox` 有 `is_boss` 标志，影响忍术投射物行为

### 3.4 剑术系统（4种方向技）

剑术是项目中最具深度的系统，包含资源管理、冷却、输入缓冲和连招：

```
按下 L 键 → SwordReadyState（急停进入架势）
  ├── 上 + L → SwordUppercut  （上挑，空中可释放，1.2倍跳跃力）
  ├── 下 + L → SwordDownslash（下劈，仅空中，快速下落）
  ├── 前 + L → SwordDash      （突进，2.5倍移速，空中可释放）
  └── 后 + L → SwordSpin      （旋斩，短距离滑行，下蹲hitbox）

每招消耗: 2 TP (冷却5秒)
必杀技: 10 TP (冷却10秒) - DragonFlash（真龙闪华）
```

**连招缓冲机制：** 在剑术动画播放期间按下 `L+方向`，输入会被缓存(0.3s)，动画结束后自动衔接下一招。这实现了类似格斗游戏的取消连段手感。

### 3.5 忍术系统（4种投射物）

| 忍术 | 伤害 | 机制 |
|------|------|------|
| 火焰 | 1/每发 | 3发扇形散射（上方向），直线飞行 |
| 火球 | 1/每发 | 3发扇形散射（下方向），直线飞行 |
| 棱刃 | 1/每帧 | 弹簧振子模型，上下弹动，锁定玩家X轴 |
| 回旋镖 | 1/每帧 | 弹簧振子模型，绕玩家回旋 |

**弹簧振子物理模型**（棱刃/回旋镖）：
```gdscript
# 核心公式：F = -k * x  (胡克定律)
acceleration = -spring_stiffness * offset
velocity += acceleration * delta
velocity *= damping  # 阻尼衰减
```

### 3.6 音频系统（三层架构）

```
SoundEventResource (.tres)     ← 音频身份证（stream, volume, pitch, bus）
        ↓
SoundRegistry (Autoload)       ← 启动时扫描 .tres 文件，建立字典
        ↓
AudioManager (Autoload)        ← 统一播放接口 play_sound("id")
  ├── BGM 专用播放器 (1个)
  └── SFX 播放器池 (4个，随机音高偏差 ±pitch_variance)
```

这是一套**数据驱动**的音频架构：策划不需要写代码，在编辑器中创建 `.tres` 资源文件即可添加新音效。

### 3.7 UI系统

[UIManager](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/ui/UIManager.gd) 是一个 `CanvasLayer` 全局 UI：

- HP/MP/TP 条形图（`scale.x` 映射比例）
- 剑术冷却遮罩（`cd_masks` 字典驱动，实时缩放遮罩高度）
- 忍术图标切换
- 浮动消息提示（"忍術不足！" / "劍術不足！"）

### 3.8 敌人系统

三种敌人类型都使用 `CharacterBody2D`：

| 类型 | 特点 |
|------|------|
| [Enemy](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/enemy/Enemy.gd) | 标准巡逻兵，地板边缘检测回头、墙碰回头、巡逻范围回头 |
| [SmallEnemy](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/enemy/SmallEnemy.gd) | 简化版 |
| [AirEnemy](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/enemy/AirEnemy.gd) | 支持飞行模式（禁用重力） |

所有敌人都支持 **EnemyData 资源覆盖**默认属性，数据驱动配置。

### 3.9 消耗品系统（Item）

[Item](file:///e:/Godot/GodotPro/NinjaGaiden_Return/scripts/item/Item.gd) 实现了四阶段生命周期：
1. **Floating**（闪烁悬浮）→ 玩家攻击触发掉落
2. **Falling**（下落）→ 重力落至地面
3. **GroundWait**（地面等待，5s）
4. **FastFlash**（快速闪烁3s后消失）

三种类型：HP恢复(8)、小MP恢复(2)、大MP恢复(16)

## 四、关键技术亮点

### 4.1 空中失衡机制（Imbalance）
当玩家在空中向**背面**方向移动时触发，移速降至 70%，动画切换为失衡姿态。这是一套打击感十足的惩罚机制，防止空中无限变向。

### 4.2 蹬墙跳锁时保护（Wall Jump Lockout）
蹬墙跳后 0.15 秒内禁止向墙方向移动，防止瞬间回吸墙面。但如果玩家主动按**相反方向**，锁时立刻清零——这是一个精妙的"防误触但不限制高手"的设计。

### 4.3 平台穿透防回吸（Regrab Cooldown）
从吊台下跳时，携带 0.25 秒冷却保护传递给 `PlatformDropState` → `FallState`，期间关闭头顶雷达，防止角色瞬间被吸回吊台。

### 4.4 无敌结束后的重叠检测
受伤无敌时间（1.5s）结束后，执行一次物理射线 `intersect_shape` 检测，如果玩家仍与敌人攻击框重叠，立即再次受伤——防止利用无敌帧站在敌人攻击范围内。

### 4.5 PAL 模式
`PALMode` Autoload 将 `Engine.time_scale` 设为 0.833，模拟 NES 欧版 50Hz 节奏，让老玩家找到熟悉的"慢一拍"手感。

## 六、项目特色总结

1. **FC原点复刻**：480×270 分辨率 + PAL模式 50Hz 节奏，精准还原FC手感
2. **自研状态机**：19个状态覆盖从基础移动到高级剑术的完整动作谱系
3. **组件解耦**：输入/移动/动画/剑术/忍术 职责分明，修改互不影响
4. **数据驱动**：PlayerData/EnemyData/SoundEventResource 均可在编辑器中配置
5. **战斗深度**：4向剑术 + 连招缓冲 + 4种忍术 + 真龙闪华必杀，操作上限高
6. **容错设计**：跳跃缓冲、无敌帧、防回吸、墙体锁时保护、动画容错检查——细节拉满
7. **弹簧振子物理**：棱刃和回旋镖使用胡克定律驱动的弹性运动，视觉效果流畅自然

这是一个结构清晰、代码风格一致、细节打磨到位的同人游戏项目。架构上最大的亮点是**组件化 + 状态机**的双层设计，使得复杂的动作游戏逻辑被清晰地分解为可管理的模块。
