## 🎮 项目全面分析报告

### 一、项目概览

这是一个 **Godot 4.x 的《忍者龙剑传》同人动作游戏**，主角是 **Ryu（隼龙）**。项目采用组件化架构，使用了大量 `.tres` 资源文件做数据驱动。

---

### 二、音频系统

音频系统由三层组成：

| 层 | 文件 | 职责 |
|---|---|---|
| 资源定义 | [SoundEventResource.gd](file:///d:/Godot/GodotProject/NinjaGaiden_Return/resources/audio/SoundEventResource.gd) | 定义单条音频的属性：event_id、stream、bus(SFX/BGM)、音量、音调、随机偏移、冷却 |
| 注册中心 | [SoundRegistry.gd](file:///d:/Godot/GodotProject/NinjaGaiden_Return/autoload/SoundRegistry.gd) | autoload，在 `_ready()` 中加载所有 `.tres` 音频资源到一个字典 |
| 播放器 | [AudioManager.gd](file:///d:/Godot/GodotProject/NinjaGaiden_Return/autoload/AudioManager.gd) | autoload，8 个 SFX 音效池 + 1 个 BGM 播放器，全局统一接口 `play_sound(event_id)` |

**帧同步音效**：[PlayerAudioComponent.gd](file:///d:/Godot/GodotProject/NinjaGaiden_Return/scripts/components/PlayerAudioComponent.gd) 通过 `FRAME_SOUNDS` 字典，在动画的特定帧自动播放音效。

**Boss 相关要点**：为 Boss 添加音效只需：
1. 创建 `SoundEventResource.tres`
2. 在 `SoundRegistry._audio_files` 数组中注册路径
3. 调用 `AudioManager.play_sound(&"boss_xxx")`

### 四、伤害系统

**伤害链路**：

```
攻击方 HitBox (Area2D, monitoring=true)
  → 检测到 HurtBox (Area2D, monitorable=true)
    → 调用 HurtBox.take_damage(damage)
      → 发射 took_damage 信号
        → 受击方 _on_took_damage(amount) 扣血
```

**关键文件**：

| 文件 | 伤害值 | 说明 |
|------|--------|------|
| [SwordHitBox.gd](file:///d:/Godot/GodotProject/NinjaGaiden_Return/scripts/components/SwordHitBox.gd#L1-L6) | `damage=2` | 普攻/剑术 |
| [DragonFlashHitBox.gd](file:///d:/Godot/GodotProject/NinjaGaiden_Return/scripts/components/DragonFlashHitBox.gd#L1-L4) | `damage=2` | 必杀技 |
| [EnemyHitBox.gd](file:///d:/Godot/GodotProject/NinjaGaiden_Return/scripts/components/EnemyHitBox.gd#L1-L17) | `damage=1` | 敌人攻击 |
| 忍术弹体 | `damage=1` | 所有忍术投射物 |

**Boss 关键 — `is_boss` 标志**：

在 [HurtBox.gd](file:///d:/Godot/GodotProject/NinjaGaiden_Return/scripts/components/HurtBox.gd#L1-L18) 中有 `@export var is_boss: bool = false`，所有忍术投射物在命中后检查此标志：

```gdscript
# 所有忍术弹体的统一逻辑
if area is HurtBox:
    area.take_damage(1)
    if area.is_boss:
        queue_free()  # Boss 不秒杀，弹体消失
```

这意味着 **Boss 不能被忍术秒杀**，普通小怪碰一下忍术弹体就死。

### 九、做 Boss 需要关注的关键点

基于以上分析，做 Boss 时你需要考虑：

1. **Boss 脚本**：建议继承或参考 `BaseEnemy`，但 Boss 通常有更复杂的状态机（多阶段），可以像 `Soldier` 一样用 `enum` 实现内部状态，也可以像玩家一样用 `StateMachine`。

2. **HurtBox 必须勾选 `is_boss=true`**：否则忍术一发秒杀。

3. **Boss HP 条**：UIManager 目前只显示玩家的 HP/MP/TP，Boss 需要单独的 HP 条（可能需要扩展 UIManager 或在 Boss 场景中独立实现）。

4. **BossGateTrigger** 已经就绪：玩家进入 Boss 区域自动关门，支持 `revert_on_exit` 离开后重新开门。

5. **BossTransitionZone** 已就绪：用于从普通关卡切换到独立 Boss 场景，支持 `fade_duration` 控制黑屏时长。

6. **Boss 音效**：遵循 `SoundEventResource → SoundRegistry → AudioManager.play_sound()` 的流程。

7. **Boss 数据**：可以继承 `BaseEnemyData` 或新建 `BossData` 资源类。

8. **Boss 击败后**：需要调用 `PlayerStateManager.reset()` 恢复玩家满状态，并触发 `SceneTransition.fade_to_scene()` 返回关卡。

9. **PALMode**：全局 `time_scale=0.833`，注意 Boss 的计时器在这个速度下运行。

10. **碰撞层级**：玩家攻击框碰撞 Enemy 层（第 5 层），敌人攻击框碰撞 Player 层（第 1 层），需要确保 Boss 的碰撞层配置正确。
