以后你每想给游戏加一个声音，只需要走这三步，完全不需要动代码：

1. 给新音效办“身份证”（Resource）
右键点击你的 resources/audio/player/ 文件夹 -> 选择 New Resource（新建资源）。

搜索并选中 SoundEventResource，点击 Create。

给文件起个名字，比如：player_jump_event.tres。

2. 在检查器（Inspector）里配置它的灵魂
双击打开这个新建的 player_jump_event.tres，在右侧面板里这样填：

Event Id：填写 p_jump（这是你在代码里呼叫它的短名字）。

Stream：把你准备好的跳跃 .wav 或 .mp3 音效拖进来。（如果没有，先继续用 testbgm.mp3 顶替，完全没问题）。

Bus：【重要改动】一定要选择 SFX！背景音乐我们选 BGM，但动作音效必须选 SFX。因为我们刚才在代码里写了，SFX 会进入“播放器池”，支持 4 个声音同时响（比如一边砍怪，一边跳跃，声音不会互相吞掉）。

Pitch Variance（核心打击感秘诀）：把它调到 0.15 左右。

💡 为什么？ 这样玩家每次跳跃时，AI 都会自动在 0.85 到 1.15 之间随机变调。声音听起来就会忽粗忽细，极具灵动感，FC 时代的很多经典动作游戏（包括忍龙、像素鸟）都用了这个技巧。

保存（Ctrl + S）。此时你可以重启一下游戏，控制台会立刻打印：【音频系统】成功登记声音：p_jump。

3. 在状态机里精准呼叫
打开你的跳跃状态脚本（如 JumpState.gd），把刚才的测试代码改成新身份证的 ID：

GDScript
func enter() -> void:
	# 换成你刚才在身份证上写的 Event Id
	AudioManager.play_sound(&"p_jump") 
保存并运行游戏，去按下跳跃键，听听看它是不是带有了随机变调的奇妙效果！

📋 附：推荐给你的《忍龙》音频命名规范
随着你游戏越做越大，声音会越来越多。建议你按照 Claude 推荐的这套规范去给你的 Event Id 命名，这样不仅你自己看着清晰，以后让 AI 帮你写逻辑时，AI 也能一眼看懂：

主角动作类：

p_jump（主角跳跃）

p_land（主角落地）

p_hurt（主角受伤惨叫）

剑术连招类：

atk_slash_1（普通攻击第 1 刀斩击声）

atk_slash_2（普通攻击第 2 刀斩击声）

atk_slash_3（普通攻击第 3 刀重劈声）

忍术类：

nin_firewheel（火龙轮出手音效）

nin_shuriken（手里剑飞出音效）

环境与怪物类：

bgm_stage_1（第一关背景音乐）

e_bat_death（蝙蝠怪物死亡音效）
