

# 🎮《忍者龙剑传：归来》项目进度与架构汇总报告

---

## 一、项目概述

| 项目 | 内容 |
|---|---|
| **游戏名称** | 忍者龙剑传：归来（Ninja Gaiden: Return） |
| **引擎** | Godot 4.6（GL Compatibility 渲染模式） |
| **游戏类型** | 横版卷轴动作平台游戏（2D Side-Scrolling Action Platformer） |
| **游戏分辨率** | 480×270（窗口缩放至 1280×720） |
| **灵感来源** | FC 原版《忍者龙剑传III》（TECMO, 1991） |
| **设计理念** | 继承忍龙之父吉泽雄秀理念，核心爽感：**一刀秒杀、快速反应、割草、怀旧** |
| **世界观** | 融合魔幻与科幻（量子科技、人工智能），架空于原版 2 代和 3 代之间 |
| **章节规划** | 共 7 大章节，每章有独立关卡标题和过场动画（剧情暂不开发） |

## 三、核心架构设计

### 3.1 整体架构：组件化 + 有限状态机（FSM）

```
┌─────────────────────────────────────────────────────────┐
│                      Player (CharacterBody2D)            │
│  ┌──────────────────────────────────────────────────┐   │
│  │               Components（组件层）                  │   │
│  │  InputComponent  │ MovementComponent              │   │
│  │  AnimationComponent │ SwordComponent (TP+CD)      │   │
│  │  NinjutsuComponent (MP+切换)                      │   │
│  └──────────────────────────────────────────────────┘   │
│  ┌──────────────────────────────────────────────────┐   │
│  │              StateMachine（状态机层）               │   │
│  │  Idle ⇄ Run ⇄ Crouch                             │   │
│  │    ↕      ↕                                        │   │
│  │  Jump ⇄ Fall ⇄ Wall ⇄ Hang ⇄ PlatformDrop       │   │
│  │    ↕      ↕                                        │   │
│  │  GroundAttack / AirAttack / CrouchAttack          │   │
│  │  GroundNinjutsu / AirNinjutsu                     │   │
│  │  SwordReady → SwordDash/Uppercut/Downslash/Spin   │   │
│  │  DragonFlash (必杀技)                              │   │
│  │  Hurt (受伤)                                       │   │
│  └──────────────────────────────────────────────────┘   │
│  ┌──────────────────────────────────────────────────┐   │
│  │              AttackRoot（攻击框层）                  │   │
│  │  HitBox / SwordDashBox / DragonFlashHitBox       │   │
│  └──────────────────────────────────────────────────┘   │
│  ┌──────────────────────────────────────────────────┐   │
│  │              HurtRoot（受击框层）                    │   │
│  │  HurtBox (可动态切换站立/下蹲尺寸)                  │   │
│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 3.2 设计模式与关键机制

| 设计模式 | 应用场景 | 说明 |
|---|---|---|
| **有限状态机（FSM）** | 玩家所有行为 | 20 个状态，`StateMachine.gd` 自动收集子节点、防错兜底初始化 |
| **组件模式** | 输入/移动/动画/剑术/忍术 | 解耦到独立 Component 节点，`Player.gd` 通过 `@onready` 引用 |
| **数据驱动** | `PlayerData` / `EnemyData` | Resource 文件配置数值，支持编辑器内可视化调整 |
| **信号通信** | 伤害/TP/MP/切换忍术 | `HurtBox.took_damage`、`SwordComponent.tp_changed` 等 |
| **碰撞层分离** | 7 个物理层 | Player / Enemy / Terrain / Platform / PlayerAttack / EnemyAttack / Item |
| **输入缓冲** | 跳跃缓冲 0.12s | 提升手感，剑术也有独立的输入缓冲和冷却系统 |

### 3.3 物理碰撞层设计

| 层 | 名称 | 用途 |
|---|---|---|
| Layer 1 | Player | 玩家本体 |
| Layer 2 | Enemy | 敌人本体 |
| Layer 3 | Terrain | 地形（实心地面/墙壁） |
| Layer 4 | Platform | 单向平台（可下穿/悬挂） |
| Layer 5 | PlayerAttack | 玩家攻击框（检测敌人受击框+消耗品） |
| Layer 6 | EnemyAttack | 敌人攻击框（检测玩家受击框） |
| Layer 7 | Item | 消耗品（检测玩家攻击框） |

### 3.4 输入映射

| 按键 | 动作名 | 功能 |
|---|---|---|
| W/A/S/D | `nav_up/down/left/right` | 移动/攀爬/下蹲 |
| K | `jump` | 跳跃（含 0.12s 输入缓冲） |
| J | `attack` | 龙剑近战攻击 |
| L | `special_move` | 剑术动作（备战→方向键出招） |
| U | `ninjutsu` | 释放当前装备的忍术 |
| I | `switch_ninjutsu` | 切换忍术（4种循环） |
| O | `finishing_move` | 必杀技：真龙华闪 |
